# 直连、公网中转、IPLC/IEPL专线机场对比指南

很多人刚接触翻墙时，都会听到一些名词：  
**直连机场、中转机场、专线机场**，但并不清楚它们到底有什么区别。

本篇内容会从最基础的概念讲起，用**尽量不偏技术**的方式，带你理解目前主流的翻墙方案，以及它们各自适合什么人使用。

## 墙是什么

![墙是什么](https://i.theojs.cn/docs/20251222114932272.webp)

“墙”指的是 **Great Firewall（GFW，防火长城）**，是一套部署在国内出口的网络审查与封锁系统。

当国内用户访问境外网站时，流量会经过运营商出口并被防火墙检测：

- 对部分网站（如 Google、YouTube 等）**直接阻断访问**
- 对异常流量进行 **深度包检测（DPI）**
- 主动探测、识别翻墙行为并进行 **封 IP / 封端口**

随着时间推移，GFW 已不只是“封网站”，而是具备了：

- 流量特征分析
- 主动探测
- 自动封锁能力

## 直连机场

![直连机场](https://i.theojs.cn/docs/20251222114932271.webp)

**直连机场**，是最早、也是结构最简单的一种翻墙方式。

原理很简单：  
在国内网络与国际互联网之间，放置一台**境外服务器**作为中转节点，用户直接连接这台服务器，再由它访问国外网站。

由于防火墙并不是“封所有国外 IP”，所以在一定时间内，这种方式是可以正常使用的。

### 特点

- 架构简单
- 成本低
- 节点 IP 暴露在公网中

一旦流量特征异常或使用人数过多，就容易被防火墙识别并封锁，只能通过**频繁更换 IP 或端口**来维持可用性。

**优点**：价格便宜  
**缺点**：速度一般、稳定性差、容易被封

## 中转机场

![中转机场](https://i.theojs.cn/docs/20251222114932270.webp)

为了解决直连机场“太容易被封”的问题，就出现了**中转机场**。

中转机场的核心思路是：  
**在国内增加一层中转节点**。

### 工作流程

1. 用户 → 国内中转服务器
2. 国内中转服务器 → 加密隧道过墙
3. 境外落地服务器 → 访问国际互联网

这样做的好处是：

- 国内入口可控、质量更好
- 过墙流量更集中、更隐蔽
- 不需要频繁更换节点 IP

### 特点

- 稳定性明显高于直连
- 延迟略高于直连
- 成本高于直连

**优点**：稳定、不容易被封  
**缺点**：价格比直连贵

## 国际专线机场（IPLC / IEPL）

![国际专线](https://i.theojs.cn/docs/20251222114932269.webp)

只要流量**经过防火墙**，理论上就存在被封锁的可能。

于是就出现了目前最稳定、但成本也最高的一种方案：  
**国际专线机场（IPLC / IEPL）**。

### 什么是国际专线

国际专线是运营商提供的 **物理级跨境专用线路**，例如：

- 深圳 → 香港
- 上海 → 日本

这些线路并不走普通公网出口，也**不经过 GFW**，而是直接通过运营商内部的跨境光纤通信。

### 特点

- 不经过防火墙
- 延迟极低
- 稳定性极高
- 成本昂贵

因此，国际专线通常用于：

- 游戏加速
- 视频流媒体
- 企业跨境办公

**优点**：速度快、稳定、不被封  
**缺点**：价格高

## 简单总结

| 类型     | 稳定性 | 延迟   | 价格 | 适合人群           |
| -------- | ------ | ------ | ---- | ------------------ |
| 直连机场 | ★☆☆    | 中     | 低   | 轻度使用           |
| 中转机场 | ★★☆    | 中偏高 | 中   | 大多数用户         |
| 国际专线 | ★★★    | 极低   | 高   | 游戏 / 视频 / 工作 |

如果你只是日常浏览、看视频，中转机场已经足够；  
如果你对稳定性和延迟要求极高，那么国际专线才是终极方案。
